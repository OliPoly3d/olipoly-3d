Production Workflow

Scheduling, closeout, and cancellations

Active work

  • Accepted jobs should have a status, due expectation, and material assumptions.
  • Printing jobs should have a printer/machine assigned when possible.
  • Post-print jobs should be checked for packaging, delivery, and finance needs.

Canceled work

  • Canceled jobs remain in history.
  • Reserved/held filament should be released.
  • Cancellation reason should explain why the job stopped.

Closeout

  • Deduct actual filament/material usage.
  • Add finished goods if applicable.
  • Confirm finance posting only when appropriate.

Regression test

  • Create test job.
  • Reserve material.
  • Cancel job and verify release.
  • Close job and verify inventory movement.
Inventory Workflow

Receive, mount, reserve, reorder

Reorder policy

Use policy to distinguish true reorder-needed materials from specialty, watch-only, seasonal, or discontinued materials.

Mounted rolls

Keep mounted/refill/spool status clean so Production can use the right material source.

Sync safety

Use Export Backup before full rebuilds. Keep movement log behavior separate from cloud item sync if ledger constraints change.

Search standards

Inventory search should work for material, color, brand, supplier, and notes.

Finance Workflow

Income, expenses, sales tax, and year-end

Income entries

Enter sale amount before tax. Choose destination county and tax rate. Sales tax collected is tracked separately from revenue.

Ohio sales tax

Use the semiannual Ohio Sales Tax Filing section for county totals and filing readiness.

Expense mapping

Use visible category as the authority for known categories such as Event Booth, Materials, Packaging, Shipping, and Marketplace Fees.

Year-end

Use Schedule C summary as a bookkeeping helper, not final tax advice.

Recovery

Backup, restore, and deployment safety

Before major changes

  • Export backups from backend pages.
  • Download repo ZIP from GitHub.
  • Commit a known-good checkpoint.

If GitHub Pages fails

  • Check GitHub Status first.
  • A successful build plus queued deployment timeout usually points to GitHub Pages infrastructure.
  • Do not keep committing repeatedly during an incident.
Quality Gate

Release checklist

Functional

  • Hub loads.
  • Search works across main modules.
  • Cloud sync does not loop.
  • No console syntax errors.

Workflow

  • Quote → Order.
  • Order → Production.
  • Production → Inventory.
  • Finance reports reconcile.

UI

  • Desktop and mobile are readable.
  • Buttons are usable.
  • Internal private pages remain noindexed.

Checkpoint

  • Commit with clear message.
  • Verify Pages deploys green.
  • Download fresh repo backup.
