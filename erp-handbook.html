ERP 1.0 Operating Architecture

One project. Multiple views.

Production Control is the operational source of truth. Quote Tool publishes the customer-facing quote. Orders Admin handles fulfillment, documents, tracker visibility, payment, and Finance Pro handoff. Inventory Control is the material ledger. Finance Pro is the sales/tax ledger.

Customer Workflow

Lead → Estimate → Quote → Order → Delivery

1. Start in Production Control

Create an estimate, burn a Q number, estimate costs, and decide the sell price. Inventory is not touched yet.

2. Publish through Quote Tool

Push the estimate into Quote Tool to produce the polished PDF/link/email. Quote Tool is the customer output layer.

3. Acceptance creates OP

Accepted Q-000123 becomes OP-000123 and moves Production Control into awaiting_design.

4. Decline/void stays clean

Declined or voided quotes keep their Q number, create no OP, touch no inventory, and move out of the active flow.

Production Workflow

Actuals, reservation, printing, QC

Actuals before reservation

  • Update design hours after design work.
  • Use slicer runtime and grams by color/material.
  • Confirm actual production plan before reserving material.

Start printing

  • Production Control changes status to printing.
  • Orders Admin mirrors In Production.
  • Public tracker becomes accurate without double entry.

QC complete

  • Prompt for failed print/scrap overage.
  • Consume reserved inventory plus overage.
  • Save production closeout snapshot.

After production

  • Production is closed/history.
  • Orders Admin resumes ownership for customer fulfillment, documents, payment, and Finance Pro.
Inventory Workflow

Receive, reserve, consume, reorder

No reserve during quote

Estimate, quote_sent, declined, and void states do not reserve material.

Reserve after actual plan

Accepted jobs reserve only when the real slicer/material plan is confirmed.

Consume at QC

Final consumption happens at QC/closeout, with scrap and failed print overages included.

Finance remains separate

Inventory tracks roll-level material cost. Finance tracks transaction-level purchases and sales.

Finance Workflow

Revenue, expenses, sales tax

Sales come from Orders Admin

When paid/complete, Orders Admin pushes revenue, tax, county, customer/order details, and payment info to Finance Pro.

No gram-level finance bridge yet

Finance Pro should not receive individual roll usage. Production keeps job profitability; Finance keeps business/tax reporting.

Purchases remain transactional

A Bambu or Amazon purchase can remain one expense entry even if inventory receives multiple rolls separately.

Future optional bridge

Later: creating inventory purchase records could optionally create Finance expenses, but that is outside ERP 1.0.

Rollback Rules

Cancellation and exception handling

Declined quote

Status becomes quote_declined. Q number remains burned. No OP, no inventory, no finance.

Void estimate

Status becomes void. Keep record for history. Do not delete or reuse Q numbers.

Cancel before printing

Cancel OP/order and release any reserved inventory if the plan had already been confirmed.

Cancel/fail during production

Prompt for material already consumed, then adjust inventory and notes so the ledger stays honest.

Quality Gate

ERP 1.0 release test path

Create Production Control estimate and confirm Q number appears.
Push Quote, send/publish, and confirm status becomes quote_sent.
Test decline/void path: no OP, no inventory movement, Q number not reused.
Accept a quote: confirm OP number matches Q number and Production wakes up in awaiting_design.
Enter actuals, confirm plan, reserve material, start printing, and verify Orders Admin/tracker mirrors status.
QC complete with scrap test. Confirm inventory consumed once and production snapshot is locked.
Finish fulfillment in Orders Admin, mark paid/complete, and push to Finance Pro.
